# 依頼分析OAシステム ― 過去データ再活用基盤 ナレッジ活用プロジェクト（oa-system-knowledge）

> 📦 本リポジトリは [`furukawa-ai/upstream-process-skills`](https://github.com/furukawa-ai/upstream-process-skills) モノレポの `upstream/projects/oa-system/` を独立リポジトリとして切り出したものです（2026/06/10、`git subtree split` で履歴保持）。**今後の更新・管理は本リポジトリで行います（管理担当: 田中利歩）。** モノレポ側ディレクトリは移管時点のスナップショットとして凍結されます。

解析センターの**依頼分析業務における過去案件データの再活用基盤**の構築を目指すプロジェクト。横浜・平塚に約20年・**約48,512件**蓄積された依頼書・分析結果・報告書（PDF／Word／Excel／PPT）が死蔵し、キーワード検索も属人化して横断活用できない状況に対し、生産性改革部 デジタルソリューション室（DS室）の2026年度「ナレッジ活用」テーマの一環として、**LLM Wiki アプローチ**による過去データの横断検索・要約基盤を提案。2026/5/14 初回打合せで **LLM Wiki 方式による小規模試行**を合意した。**2026/6/5（金）に先方担当者より試行用データを受領**（2015年度分から抜粋した **37件分の依頼フォルダ・計377ファイル**）し、ファイル内容の確認・LLM Wiki 生成に着手。約2週間後（**6/19 前後**）に状況共有の打合せを予定している。

- **対象部門**: 解析センター（科学分析技術課ほか）の依頼分析業務。現行依頼分析処理システム担当＝飯塚氏
- **対象業務**: 外部・社内からの分析依頼受付 → 分析実施 → 報告書作成。過去案件（依頼書・分析手法・結果・報告書）の参照・再活用
- **先方の中核課題**: 約48,512件の過去依頼分析データが死蔵。キーワード検索は実装済だが担当者ごとに任意設定で属人化し、自然言語・依頼書ファイルでの横断検索ができない
- **ステータス**: **Phase 1.5（LLM Wiki 方式 試行施策）― 試行用データ受領済み、内容確認・ウィキ生成に着手**（次回：6/19 前後に状況共有打合せ）
- **DS室担当**: 田中利歩（主担当・データ解析ソリューション課）／市川祐司／川邊翔吾／力宗真寛（LLM Wiki）
- **先方担当**: 飯塚健児 氏（解析センター 科学分析技術課・依頼分析処理システム担当）
- **関係者**: 金丸知史 氏（CPO室企画課・社内AI推進ハブ／つなぎ役。本案件のつなぎ役で進捗報告を受ける立場）
- **テーマ位置づけ**: 生産性改革部 デジタルソリューション室 2026年度「ナレッジ活用」テーマの一環（相談受領 → 初回打合せ → LLM Wiki 試行合意 → 試行用データ受領・試行着手）

## LLM Wiki アプローチの妥当性

依頼分析データは「依頼書（PDF）＋分析手法＋結果＋報告書（Word/Excel/PPT）」が案件単位でまとまっており、案件ごとのナレッジ構造が明確。AI が生データから自然言語のウィキを自動生成・構造化し、それをソースとして回答する LLM Wiki 方式は、属人的キーワードに依存しない横断検索・要約と相性が良い。RAG（ベクトル検索）と異なり追加データ投入時は差分＋整合性調整のみで済み、全体再処理が不要なため運用コストを抑えやすい。先行する [functional-resin-mfg-qa](https://github.com/furukawa-ai/upstream-process-skills/tree/main/upstream/projects/functional-resin-mfg-qa)・[laser-processing-knowledge](https://github.com/furukawa-ai/upstream-process-skills/tree/main/upstream/projects/laser-processing-knowledge)・[at-design-review-knowledge](https://github.com/furukawa-ai/upstream-process-skills/tree/main/upstream/projects/at-design-review-knowledge) と同方式。

## プロジェクトの現在地

- **経緯**: 2026/4/30 相談受領（pptx）→ 2026/5/14 初回打合せ（LLM Wiki 方式・小規模試行を合意）→ **2026/6/5 試行用データ受領（2015年度分37件／377ファイル）** → 内容確認・LLM Wiki 生成に着手 → 6/19 前後に状況共有打合せ
- **採用方針**: LLM Wiki アプローチ（Claude Code 社内ローカル実装。生データから AI が Wiki を自動生成・構造化）。当初想定の「依頼書＋報告書 50件規模」に対し実受領は37件・377ファイル
- **本件の狙い**: 約48,512件の死蔵データを、自然言語や依頼書ファイルそのものを入力に横断検索し、分析手法・結果の要約付きで提示できる状態をつくること
- **次のマイルストーン**: 受領377ファイルの内容確認 → LLM Wiki 生成・評価（生成工数・コスト・出力品質・類似案件抽出精度）→ 6/19 前後の打合せで状況共有 → 全48,512件への拡張可否・本格展開方式（部分展開／RAG併用等）を判断

## 重要な制約・留意点

- **2026年度「ナレッジ活用」テーマの一環**: DS室主導で進めるテーマ。Claude Code を活用し従来の Copilot では難しかった範囲を射程に
- **データガバナンス**: 試行データに顧客情報・輸出管理データが混在する可能性があり、内容確認時に外部送信可否・マスキング要否を点検する必要がある（→ [07_初回打合せ議事録.md](project-docs/07_初回打合せ議事録.md) §6 持ち越し事項）
- **持ち越しの確認事項**: 初回打合せ時点で未確認の事項（データ実態・機密区分混在・既存IT環境・利用要件 等）が残る。LLM Wiki 試行と並行し、特に機密区分混在は試行用データ選定時に飯塚氏と整合する（経緯は [07_初回打合せ議事録.md](project-docs/07_初回打合せ議事録.md) §6）
- **社内展開**: 力宗氏作成の LLM Wiki 解説資料は再配布自由。本案件を「LLM Wiki を用いたソリューション提案事例」として社内 AI プロモーター（約100名規模）へ展開可（初回打合せで合意）
- **進捗共有**: 試行結果がまとまり次第、金丸知史 氏（CPO室企画課）へサマリーを共有
