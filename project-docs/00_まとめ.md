## 主要な論点（次回打合せで決めたいこと）

1. **LLM環境の選定**: 社内Azure OpenAI ／ Anthropic API ／ ローカルOSS LLM
2. **UI形態の選定**: 内製Webアプリ ／ Copilot Studio ／ ServiceNow組込
3. **PoC範囲の絞り込み**: 1拠点・直近3年か、全期間サンプリングか
4. **LLM Wiki構築の同時実施可否**: 検索基盤と並行か、フェーズ2以降か
5. **OCRエンジンの選定**: Azure Document Intelligence ／ Tesseract ／ PoC比較

## ご提案いただいた「Claude Code でのLLM Wiki／インデックス化」案への結論（先出し）

- **方向性は正しい**。RAG＋メタデータ抽出＋Wiki化は本テーマの王道
- ただし **Claude Code（CLIツール）を業務担当者の検索UIにするのは不適**
- **位置づけを切り分ける**：
  - 業務担当者 → Web検索UI（自然言語入力＋ファイルアップロード）
  - 開発者・運用者 → Claude Code／Agent SDK でインデックス構築・メタデータ抽出パイプラインを開発・運用
  - 各案件のサマリーをMarkdown化して社内Wikiに展開する案は強く推奨
- **データ送信先のガバナンス**は最優先で確定が必要（顧客情報／輸出管理データ混在の可能性）

詳細は [03_ソリューション提案.md §5](03_ソリューション提案.md) を参照。
