## 確認したい論点（再掲）

### 1. 制約条件

- 機密／輸出管理／顧客名秘匿対象の混在状況
- 利用可能な社内LLM環境（Azure OpenAI／その他）
- ServiceNow計画とのスコープ切り分け

### 2. データ実態

- 物理保管場所とプログラムアクセス手段
- 画像PDF比率
- 報告書テンプレートの統一性
- 平塚 2015/4以降の件数（PPTX未記載）
- 1案件あたりの平均ページ数

### 3. 利用シーン

- 想定利用者（担当者／依頼者／新人）
- 「分析手法・結果の要約」の理想的な表示イメージ
- 検索結果カードの必須項目

### 4. プロジェクト推進

- 体制（先方の担当者・コミットメント）
- 予算枠とスケジュール感
- PoC範囲の絞り込み方針

## 当課から事前にお伝えしたいこと

1. 本テーマは **典型的なRAGの社内導入** であり、当課には機能樹脂営業部1課での類似案件（技術問い合わせRAG）の進行実績がある
2. 48,512件規模は大きいが、フェーズを切れば段階的に取り組める
3. **業務プロセス改善（受付自動化）と検索基盤は分離して進めることを推奨**。検索基盤を先行できる
4. **「Claude Code を担当者UIにする」のは不適だが、開発・パイプライン構築の道具としては有効**。位置づけを整理した上で採用を提案する
